# 图表检查模块

## 检查目标

确保所有 Table 和 Figure 的 caption 信息完整，读者无需阅读正文即可理解图表内容。

## Table 检查清单

### Caption 必要元素

1. **内容概述**：表格展示什么内容
2. **数据说明**：
   - 各列/行指标的含义
   - 数值单位（%、秒、MB等）
   - 数据来源（如适用）
3. **标记说明**：
   - 加粗数字的含义（通常是最佳结果）
   - 下划线的含义（通常是次佳结果）
   - 其他特殊标记（†, *, ‡等）
4. **对比说明**：
   - baseline 来源
   - 是否是论文复现还是引用原文数字

### 常见问题

| 问题 | 示例 | 严重程度 |
|------|------|----------|
| 无指标说明 | "Results on Dataset X" 但未说明各列是什么指标 | 🔴 Critical |
| 无单位 | 数字是百分比还是绝对值不明确 | 🟡 Major |
| 标记未解释 | 加粗数字但 caption 未说明含义 | 🟢 Minor |
| 数据来源不明 | baseline 数字不知道是复现还是引用 | 🟡 Major |

## Figure 检查清单

### Caption 必要元素

1. **图的主题**：这张图展示什么
2. **子图说明**：(a)(b)(c)各是什么
3. **坐标轴**：
   - X轴代表什么
   - Y轴代表什么
   - 单位
4. **图例**：各颜色/线型代表什么
5. **特殊标注**：箭头、虚线等的含义

### 图类型特定要求

| 图类型 | 额外要求 |
|--------|----------|
| 架构图 | 各模块功能说明、数据流向 |
| 折线图 | 坐标轴含义、各线代表什么 |
| 柱状图 | 分组含义、误差棒说明（如有） |
| 热力图 | 颜色映射含义、数值范围 |
| 案例分析图 | 输入输出的具体含义 |

## 检查流程

```
1. 遍历所有 Table 和 Figure
2. 对每个图表：
   a. 阅读 caption
   b. 对照检查清单逐项核对
   c. 记录缺失信息
3. 汇总问题
```

## 输出模板

```markdown

**状态**: [✅ 通过 / ⚠️ 发现问题]

### Tables
| 表号 | Caption概述 | 指标说明 | 单位 | 标记解释 | 问题 |
|------|------------|---------|------|---------|------|

### Figures  
| 图号 | 类型 | 主题说明 | 坐标轴 | 图例 | 问题 |
|------|------|---------|--------|------|------|

### 详细问题
[对每个问题的具体描述和建议]
```
