# 相关工作检查模块

## 检查目标

通过"顺藤摸瓜"策略，检查论文是否遗漏重要相关工作。

## 检查策略

### 顺藤摸瓜法

基于论文已引用的关键文献，向外扩展搜索：

```
论文引用的关键文献
        │
        ├── 这些文献被谁引用了？（citing papers）
        │   └── 近2-3年的高引论文
        │
        └── 这些文献引用了谁？（referenced papers）
            └── 被多篇关键文献共同引用的论文
```

### 执行步骤

1. **提取关键引用**
   - 从 Related Work section 提取核心引用（通常5-10篇）
   - 优先选择：近3年、高相关性、同领域顶会

2. **扩展搜索**
   - 对每篇关键引用，搜索其近期被引论文
   - 关注近2-3年发表的工作
   - 关注同领域顶会（ACL/EMNLP/NAACL, NeurIPS/ICML/ICLR, CVPR/ICCV等）

3. **筛选候选**
   - 筛选与论文主题高度相关的工作
   - 特别关注：方法类似、问题相同、同一数据集

4. **判断遗漏**
   - 候选论文是否在论文中被讨论？
   - 如果没有，是否应该讨论？

## 遗漏类型

### 必须讨论的遗漏

| 类型 | 描述 | 严重程度 |
|------|------|----------|
| 直接竞争者 | 解决相同问题的近期方法 | 🔴 Critical |
| 方法先驱 | 提出类似核心idea的早期工作 | 🔴 Critical |
| 同数据集SOTA | 在同一数据集上的最新最好结果 | 🔴 Critical |
| 重要baseline | 领域公认的重要baseline | 🟡 Major |

### 建议讨论的遗漏

| 类型 | 描述 | 严重程度 |
|------|------|----------|
| 相关技术 | 使用类似技术但不同任务 | 🟢 Minor |
| 综述论文 | 领域重要综述 | 🟢 Minor |
| 并行工作 | 同期进行的类似研究 | 🟡 Major |

## 搜索关键词构建

基于论文内容构建搜索关键词：

```
关键词来源：
├── 论文标题中的核心词
├── 方法名称
├── 任务名称
├── 数据集名称
└── 核心技术（如 LLM, VQ-VAE, attention）

搜索模板：
- "[任务名] [年份] [会议]"
- "[方法类型] [应用领域] survey"
- "[数据集名] benchmark [年份]"
```

## Subagent Prompt

```
你是一个学术文献调研助手。

任务：检查论文的相关工作是否完整。

论文信息：
- 标题: [title]
- 任务: [task]
- 核心方法: [method]
- 数据集: [datasets]

论文已引用的关键文献：
1. [Author et al., Year] Title
2. ...

执行步骤：
1. 对每篇关键文献，搜索其近期被引论文（2023-2026）
2. 筛选与本论文高度相关的工作
3. 检查这些工作是否在论文中被讨论
4. 列出可能遗漏的重要相关工作

返回格式：
| 可能遗漏的论文 | 作者 | 年份/会议 | 相关性说明 | 遗漏严重程度 |
```

## 输出模板

```markdown
## 相关工作检查结果

**状态**: [✅ 覆盖完整 / ⚠️ 可能有遗漏]

### 检查范围
- 分析的关键引用数: X
- 搜索的时间范围: 2023-2026
- 关注的会议/期刊: [列表]

### 关键引用分析
| # | 关键引用 | 被引论文数 | 筛选后相关论文数 |
|---|---------|-----------|-----------------|

### 可能遗漏的重要工作

#### 🔴 强烈建议讨论
| 论文 | 作者 | 年份/会议 | 相关性说明 |
|------|------|----------|-----------|

#### 🟡 建议讨论
| 论文 | 作者 | 年份/会议 | 相关性说明 |
|------|------|----------|-----------|

#### 🟢 可考虑讨论
| 论文 | 作者 | 年份/会议 | 相关性说明 |
|------|------|----------|-----------|

### 遗漏原因分析
[分析为什么可能遗漏这些工作：时间原因、领域差异等]
```

## 注意事项

1. **时效性**：相关工作检查对时效性要求高，需使用最新搜索结果
2. **并行工作**：同期投稿的工作可能互相不知道，这种遗漏可以理解
3. **领域边界**：跨领域工作可能被遗漏，但不一定是问题
4. **arXiv论文**：未正式发表的arXiv论文遗漏严重程度较低
5. **搜索限制**：搜索可能不完整，标记为"可能遗漏"而非"确定遗漏"

## Limitation 检查（附带）

在检查相关工作时，同时检查：

### Limitation Section
- [ ] 是否有专门的 Limitation section 或段落
- [ ] 讨论的局限性是否真实（不是敷衍）
- [ ] 是否讨论了方法的适用范围
- [ ] 是否讨论了潜在的失败情况

### 输出格式
```markdown
## Limitation 检查

**状态**: [✅ 有充分讨论 / ⚠️ 讨论不足 / ❌ 无讨论]

**位置**: [Section X.X / 无]

**讨论的局限性**:
1. ...
2. ...

**可能遗漏的局限性**:
1. ...
```
