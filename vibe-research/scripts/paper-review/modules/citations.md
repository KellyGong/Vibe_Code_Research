# 引用检查模块

## 检查目标

验证论文中每条引用的真实性和上下文准确性。

## 检查维度

### 1. 存在性验证

**检查内容**：
- 论文是否真实存在
- 作者名是否正确
- 发表年份是否正确
- 会议/期刊名称是否正确
- 标题是否正确

**验证方法**：
1. 使用 WebSearch 搜索论文标题
2. 查找官方出版页面（ACL Anthology, arXiv, IEEE Xplore等）
3. 核对 BibTeX 信息

**结果分类**：
- ✅ 确认存在：找到官方出版记录，信息匹配
- ⚠️ 信息有误：论文存在但部分信息（年份、会议等）有误
- ❓ 待确认：无法找到但不能确定不存在
- ❌ 不存在：有充分证据表明论文不存在

### 2. 上下文恰当性

**检查内容**：
- 论文中对该引用的描述是否准确
- 是否存在曲解或误引
- 引用是否与上下文相关

**常见问题**：
| 问题类型 | 描述 | 示例 |
|----------|------|------|
| 张冠李戴 | 把A论文的方法说成B论文的 | "X et al. proposed attention mechanism" 但X并非attention首创者 |
| 过度简化 | 对方法的描述过于简化导致失真 | "prior methods simply use concatenation" 但实际方法更复杂 |
| 选择性引用 | 只引用支持自己观点的部分 | 引用论文说A方法效果差，但忽略该论文也指出B方法效果好 |
| 错误归因 | 把某概念/方法归因到错误的论文 | 术语首创者错误 |

## 分批策略

当引用数量较多（>20）时，分批验证：

```
总引用数 N
├── subagent-A: 引用 1 到 N/2
└── subagent-B: 引用 N/2+1 到 N
```

每个 subagent 独立验证，最后汇总。

## Subagent Prompt

```
你是一个学术引用核验助手。

任务：验证以下引用的真实性和上下文准确性。

引用列表:
1. [Author et al., Year] Title - Venue
   论文中描述: "..."
   
2. ...

对于每条引用：
1. 使用 WebSearch 搜索论文标题，验证是否存在
2. 核对作者、年份、会议/期刊名称
3. 如果可能，检查论文中对该引用的描述是否准确

返回格式：
| # | 引用 | 存在性 | BibTeX正确 | 上下文恰当 | 问题描述 |
|---|------|--------|-----------|-----------|----------|
```

## BibTeX 常见错误

| 错误类型 | 描述 | 示例 |
|----------|------|------|
| 年份错误 | arXiv年份 vs 正式发表年份 | 2023 arXiv 但写成 2024 ICML |
| 会议错误 | 会议名称不对 | NeurIPS 写成 NIPS（已更名） |
| 页码错误 | 页码不存在或超范围 | pages 9116-9124 但会议论文集没有这么多页 |
| 作者顺序 | 作者顺序错误 | |
| 标题错误 | 标题有typo或不完整 | |

## 注意事项

1. **搜索策略**：先搜标题，再搜作者+年份，最后搜关键词
2. **arXiv论文**：验证arXiv ID是否正确，论文是否后来发表在会议
3. **误报处理**：如果搜索未找到但论文可能确实存在（小众venue），标记为"待确认"
4. **时间限制**：每条引用验证时间控制在合理范围，避免无限搜索
